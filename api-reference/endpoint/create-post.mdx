---
title: "Create post"
api: "POST /posts"
description: "Create a new post with content for one or more social platforms."
---

## Request

<ParamField body="platforms" type="array" required>
  An array of platform entries. Each entry specifies a social account and the content to publish.
  <Expandable>
    <ParamField body="social_account_id" type="string" required>
      The UUID of the social account to post to.
    </ParamField>
    <ParamField body="content_type" type="string" required>
      The content type for this platform. Must match the social account's platform. Values: `linkedin_post`, `linkedin_carousel`, `linkedin_page_post`, `linkedin_page_carousel`, `x_post`, `facebook_post`, `facebook_reel`, `facebook_story`, `instagram_feed`, `instagram_reel`, `instagram_story`, `tiktok_video`, `youtube_short`, `threads_post`, `pinterest_pin`, `pinterest_video_pin`, `pinterest_carousel`, `bluesky_post`, `mastodon_post`.
    </ParamField>
    <ParamField body="content" type="string">
      The post content text.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="scheduled_at" type="string">
  Schedule the post for a future date and time. Format: `Y-m-d H:i:s`. Must be in the future.
</ParamField>

<ParamField body="status" type="string">
  The post status. One of: `draft`, `scheduled`, `publishing`. Defaults to `draft`.
</ParamField>

## Response

Returns the created post with its platform entries.

<RequestExample>
```bash
curl -X POST https://app.trypost.it/api/posts \
  -H "Authorization: Bearer tp_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "platforms": [
      {
        "social_account_id": "b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e",
        "content_type": "linkedin_post",
        "content": "Hello from the TryPost API!"
      }
    ],
    "scheduled_at": "2025-01-15 10:00:00",
    "status": "scheduled"
  }'
```
</RequestExample>
